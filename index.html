Query Parameters

Query parameters allow you to pass data through URLs using the ? separator. Chain multiple parameters with &.

Format: page.html?key=value&key2=value2

Real-world use cases:

  • Search: ?q=javascript&page=1
  • Filtering: ?category=tech&sort=date
  • Tracking: ?utm_source=email&utm_campaign=spring
  • Authentication: ?token=abc123&redirect=/dashboard

Anchor Links

Anchor links (fragments) allow smooth navigation to specific sections using #section-id. They can be combined with query parameters.

Format: page.html#section-id

Combined: page.html?search=term#results

Anchors use the id attribute on elements and work seamlessly with browser history.

URL Structure Breakdown

  • protocol:// ‚Äî https or http (secure or standard)
  • example.com ‚Äî Domain name
  • /path/page.html ‚Äî File location (path)
  • ?query=value ‚Äî Query string (parameters)
  • #section ‚Äî Anchor/Fragment identifier

Best Practices

  • Be descriptive: Use clear, meaningful parameter names
  • URL encode: Encode special characters (e.g., spaces as %20)
  • Avoid duplicates: Don't repeat information in path and query
  • Keep it short: Shorter URLs are easier to share and remember
  • Maintain consistency: Use lowercase, hyphens for readability
  • Document parameters: Make it clear what each parameter does
